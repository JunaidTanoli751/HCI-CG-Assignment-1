Usability Analysis: Pakistani Digital Platforms

Introduction: Why Analyze Local Apps?

Pakistan's digital ecosystem has exploded in recent years. From e-commerce giants like Daraz to ride-hailing services like Careem and digital wallets like JazzCash, millions of Pakistanis interact with these platforms daily. Understanding their UX design‚Äîwhat works and what doesn't‚Äîprovides valuable insights into practical HCI principles.

This analysis examines four major Pakistani platforms through Don Norman's design principles, identifying strengths, weaknesses, and opportunities for improvement.

Case Study 1: Daraz Mobile App

Platform Overview

Daraz App Quick Facts
Platform Type E-commerce Mobile Application
Primary Users Online shoppers across Pakistan (18-45 age group)
Key Functions Product browsing, searching, ordering, payment, tracking
User Base 10+ million active users

Usability Strengths

Clear Visual Hierarchy
Categories are prominently displayed with recognizable icons. Users can quickly identify product sections (Electronics, Fashion, Groceries) without confusion. This demonstrates strong signifiers.
Consistent Feedback
Adding items to cart triggers immediate visual confirmation with animation and a subtle cart icon update. Users always know when an action succeeds, following Norman's feedback principle.
Search Affordance
The search bar has a magnifying glass icon and placeholder text "Search in Daraz" making its function immediately obvious‚Äîexcellent affordance design.
Progressive Disclosure
Product details are revealed gradually: thumbnail ‚Üí expanded view ‚Üí full specifications. This prevents information overload while maintaining access to detailed data when needed.

Identified Problems

Usability Issues in Daraz App
Issue Norman Principle Violated Impact Severity
Too many notifications Feedback (excessive) Users disable notifications entirely, missing important updates Medium
Flash sale timer confusion Conceptual Model Users unsure if they can still order after timer shows "00:00" High
Filter reset without warning Constraints / Error Prevention Users accidentally lose 5+ applied filters when navigating back High
Inconsistent "Add to Cart" button Consistency Button location changes between product types (right vs center) Low

Recommendations

  1. Smart Notification Management: Group similar notifications, implement intelligent frequency capping, and allow granular control over notification types.
  2. Clear Flash Sale States: Show distinct states: "Active" (green), "Ending Soon" (orange), "Ended" (gray) with explicit text, not just timer countdown.
  3. Filter Preservation: Add confirmation dialog: "You have 5 active filters. Clear them?" before navigation that would reset filters.
  4. Standardize CTAs: Always place "Add to Cart" button in bottom-right position across all product types for muscle memory consistency.

Case Study 2: Careem Ride Booking Interface

Interface Analysis

Careem (now Uber Careem) revolutionized transportation in Pakistan. Its interface must work flawlessly under pressure‚Äîwhen users need rides urgently, often in unfamiliar locations.

Excellence in UX Design

Feature Design Principle Why It Works
Location pin dragging Direct Manipulation Visual, immediate feedback as pin moves; feels natural and intuitive
"Book for Now" vs "Schedule" Clear Mapping Obvious distinction between immediate and future bookings
Fare estimate before booking Transparency / Trust Removes uncertainty; users know cost upfront
Captain photo and rating Safety Signifier Builds trust before ride begins; enables accountability
Real-time captain location Continuous Feedback Reduces anxiety; users see progress without asking "where are they?"

Pain Points

‚ö†Ô∏è Problem 1: Location Name Ambiguity
Description: Pakistan has duplicate street names across cities. "Main Boulevard" exists in Lahore, Islamabad, Karachi. App sometimes selects wrong city's location.
Violated Principle: Error Prevention
Solution: Always show city name with location. Before booking, confirm: "Pickup from Main Boulevard, Lahore‚Äîis this correct?"
‚ö†Ô∏è Problem 2: Cancel Button Placement
Description: After booking, "Cancel Ride" button is too easy to tap accidentally while checking ride details.
Violated Principle: Constraints
Solution: Require two-step cancellation: tap "Cancel" ‚Üí confirm with reason selection. This prevents accidental cancellations.
‚ö†Ô∏è Problem 3: Surge Pricing Communication
Description: Small orange "1.5x" multiplier is easy to miss. Users shocked by final fare.
Violated Principle: Visibility
Solution: Large, unmissable alert: "High demand! Fare is 1.5x normal. Estimated cost: Rs 450" before booking confirmation.

Case Study 3: JazzCash Payment Flow

Context: Digital Payments in Pakistan

JazzCash pioneered mobile wallets in Pakistan, serving users from tech-savvy millennials to first-time smartphone users. Its interface must balance simplicity with functionality while maintaining absolute security.

Send Money Flow Analysis

Task: Send Rs 2,000 to a friend's mobile number

Current User Journey (7 Steps)
  1. Open app ‚Üí Enter 4-digit PIN
  2. Tap "Send Money" from home screen
  3. Select "Mobile Account" (vs Bank Account)
  4. Enter recipient number: 03XX-XXXXXXX
  5. Enter amount: 2000
  6. Review transaction details
  7. Confirm with PIN again
Strengths
  • Double PIN verification prevents unauthorized transactions
  • Review screen shows all details before confirming
  • Amount keypad only shows numbers (constraint prevents text entry)
  • Success confirmation shows transaction ID and receipt option
Critical Issues
Problem User Impact Frequency Norman Principle
No contact list integration Must memorize/manually type 11-digit numbers Every transaction Efficiency
Unclear error messages "Transaction failed - Error 4092" (what does this mean?) ~12% of attempts Feedback Quality
No recent recipients Re-enter same friend's number repeatedly 70% repeat recipients Efficiency / Memory
Amount button confusion "2000" looks like "20.00" on small screens Occasional Visibility
Redesign Recommendations
üì± Add Contact Picker
Allow selecting from phone contacts. Show contacts who already use JazzCash with verified checkmark. Reduces typing errors and speeds up transactions.
üí¨ Human-Readable Errors
Instead of "Error 4092" show: "Insufficient balance. You have Rs 1,500 but tried to send Rs 2,000. Please add money or send a smaller amount."
üïê Recent/Favorite Recipients
Show 5 most recent recipients on "Send Money" screen. Add star icon to favorite frequent recipients. Both features dramatically reduce interaction time.
üí∞ Clear Amount Display
Format as "Rs 2,000.00" with clear rupee symbol and thousand separator. Add confirmation: "You're sending Two Thousand Rupees" in Urdu/English for verification.
Security vs Usability Balance

"The most secure system is one no one can use. The most usable system might have security vulnerabilities. Great design finds the balance."

JazzCash correctly prioritizes security (double PIN, review screen) but sometimes at the cost of efficiency. The proposed changes maintain security while improving usability‚Äîthe hallmark of excellent UX design.

Case Study 4: Foodpanda Restaurant Selection

Scenario: Ordering Food for Iftar

Time-sensitive context: User wants to order food for Iftar (sunset meal during Ramadan). They have 45 minutes before Iftar time. Interface must help them make quick, confident decisions.

What Works Well

  1. Filter by delivery time: "Delivers in 30 mins" badge on each restaurant helps users quickly identify feasible options
  2. Visual food photos: High-quality images reduce uncertainty about what you're ordering
  3. Real reviews with photos: Customer photos show actual received food (vs idealized menu photos)
  4. Minimum order indicator: Clear "Min. Rs 299" prevents cart abandonment at checkout
  5. Live order tracking: Map view with rider location reduces "where's my food?" anxiety

UX Friction Points

Obstacles in User Journey
Stage Problem User Frustration Fix
Browse Too many filter options "I just want biryani, why do I need 12 filters?" Smart defaults: If search "biryani," auto-apply Pakistani cuisine filter
Select Customization overload 15+ checkboxes for pizza toppings creates decision paralysis Offer "Popular combos" + custom option for power users
Checkout Hidden delivery charges "Cart showed Rs 800, checkout shows Rs 950!" Show delivery fee on restaurant page, not just at checkout
Payment Cash/Card toggle buried Users miss preferred payment method, have to restart Prominent payment method selection before order review
Track Estimated time keeps changing "First said 30 mins, now saying 50 mins" Show range: "30-40 mins" instead of false precision

Key Insights Across All Platforms

Common Strengths in Pakistani Apps

  • Visual design: All apps use clear icons and imagery suitable for diverse literacy levels
  • Localization: Support for Urdu language and local payment methods (COD, Easypaisa, etc.)
  • Offline resilience: Apps handle poor network conditions better than global counterparts
  • Price transparency: Pakistani users highly value seeing costs upfront

Common Weaknesses

  • ‚ö†Ô∏è Error messages: Often technical jargon instead of helpful guidance
  • ‚ö†Ô∏è Notification overload: Aggressive marketing reduces trust in important alerts
  • ‚ö†Ô∏è Hidden costs: Delivery/service charges revealed too late in process
  • ‚ö†Ô∏è Complex authentication: Too many verification steps frustrate users

Conclusion: Lessons for Future Designers

Analyzing these four Pakistani platforms reveals that good UX isn't about following Western design trends‚Äîit's about understanding your users deeply:

  1. Context matters: Designing for Karachi traffic is different from designing for NYC traffic
  2. Error prevention saves support costs: Every prevented mistake saves customer service interactions
  3. Trust is fragile: Hidden fees or confusing errors damage long-term user relationships
  4. Accessibility isn't optional: Diverse user base requires inclusive design from day one
  5. Test with real users: Assumptions about user behavior are often wrong

"The best interfaces disappear. Users shouldn't think about the app‚Äîthey should focus on ordering their biryani, booking their ride, or sending money to family. When design gets out of the way, technology truly serves humanity."
