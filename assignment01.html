Sample Exam Questions & Answers

The following questions are typical of what you might encounter in the exam. Study these carefully as they test fundamental HTML and HCI concepts.

Section A: HTML Fundamentals

Question 1: Semantic HTML

Q: Explain the difference between <div> and <section>. When should each be used?

A:

  • <div> is a generic container with no semantic meaning. It should be used only when no other semantic element is appropriate, typically for styling or JavaScript purposes.
  • <section> represents a thematic grouping of content, typically with a heading. It should be used for distinct sections of a document that would appear in the document outline.
  • Example: Use <section> for chapters, tabbed content, or different topic areas. Use <div> for wrapper elements purely for layout.

Question 2: Form Accessibility

Q: Why is it important to use <label> elements with form inputs? Provide an example.

A:

Labels are crucial for:

  1. Accessibility: Screen readers announce the label when the input receives focus
  2. Usability: Clicking the label focuses the associated input
  3. Clarity: Makes forms easier to understand

Example:

<label for="email">Email Address:</label>
<input type="email" id="email" name="email" required>

Question 3: Table Structure

Q: What is the purpose of <thead>, <tbody>, and <tfoot> in tables?

A:

<thead>
Contains header rows with column headings. Helps screen readers identify headers and can be repeated on printed pages.
<tbody>
Contains the main data rows of the table. Separates data from headers and footers for better styling and semantics.
<tfoot>
Contains footer rows with summary information (totals, averages, etc.). Appears at bottom visually but can be defined anywhere in code.

These elements improve accessibility, enable better styling, and make tables easier to maintain.

Question 4: Image Alt Text

Q: Write appropriate alt text for: (a) a decorative border image, (b) a photo of Don Norman, (c) a chart showing website traffic data.

A:

  • (a) Decorative: alt="" (empty alt text, screen readers will skip it)
  • (b) Photo: alt="Portrait of Don Norman, author of The Design of Everyday Things"
  • (c) Chart: alt="Bar chart showing website traffic increasing from 1000 visitors in January to 5000 in December 2025"

Key principle: Alt text should convey the same information as the image. Decorative images get empty alt; informative images get descriptive text.

Question 5: Document Structure

Q: Write the complete HTML5 boilerplate structure including DOCTYPE, html tag with language, head section with charset and viewport, and body.

A:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    <!-- Content goes here -->
</body>
</html>

Section B: HCI Concepts

Question 6: Norman's Affordances

Q: Define "affordance" and provide three examples from everyday objects or digital interfaces.

A:

Affordance is a property of an object that suggests how it should be used. It's the relationship between an object and a person that determines how the object could be used.

Examples:

  1. Door handle (physical): A horizontal bar affords pushing; a vertical handle affords pulling
  2. Button (digital): A raised, clickable area affords pressing/clicking
  3. Scroll bar: Vertical bar with arrows affords dragging to navigate content

Question 7: Feedback Principle

Q: Explain why feedback is important in interface design. Give an example of poor feedback and how to improve it.

A:

Importance: Feedback informs users about the result of their actions, confirms operations were successful, and helps prevent errors by keeping users informed of system status.

Poor Example: Submit button on web form that doesn't provide any indication whether form was submitted successfully.

Improvement:

  • Disable button after click to prevent double submission
  • Show loading spinner during processing
  • Display success message: "Form submitted successfully!"
  • Or show error message with specific problems if submission fails

Question 8: KLM Analysis

Q: Calculate the KLM time for the following task: User moves mouse to a button (pointing), clicks it (button press), then types a 10-character password (keystrokes). Use standard KLM times.

A:

Breaking down the task:

  1. Mental preparation: M = 1.35s
  2. Point to button: P = 1.10s
  3. Click button: B = 0.10s
  4. Home to keyboard: H = 0.40s
  5. Type 10 characters: 10K = 10 × 0.20 = 2.00s

Total time: 1.35 + 1.10 + 0.10 + 0.40 + 2.00 = 4.95 seconds

Question 9: Mapping

Q: What is "mapping" in design? Provide an example of good mapping and poor mapping.

A:

Mapping refers to the relationship between controls and their effects. Good mapping creates an intuitive connection between the control and its result.

Good Mapping: Steering wheel in a car – turn left to go left, turn right to go right. The control direction matches the result direction.

Poor Mapping: Four burners on a stove arranged in a square, but controls arranged in a single row. Unclear which knob controls which burner.

Better Solution: Arrange controls in same spatial layout as burners, or label clearly.

Question 10: GOMS Model

Q: Explain each component of GOMS (Goals, Operators, Methods, Selection rules) with an example of saving a document.

A:

Goals
What the user wants to accomplish. Example: "Save the current document"
Operators
Basic actions available to the user. Example: Click mouse, press key, move cursor
Methods
Sequences of operators to achieve goals. Example methods to save:
  • Method 1: Click File menu → Click Save
  • Method 2: Press Ctrl+S keyboard shortcut
  • Method 3: Click Save icon in toolbar
Selection Rules
Rules for choosing between methods. Example: "If hands are already on keyboard, use Ctrl+S. If using mouse, use toolbar icon or File menu."

Section C: Practical Application

Question 11: Form Design

Q: Design an accessible login form with username and password fields. Include proper labels, input types, and a submit button.

A:

<form action="/login" method="post">
    <fieldset>
        <legend>Login</legend>
        
        <p>
            <label for="username">Username:</label><br>
            <input type="text" id="username" name="username" required>
        </p>
        
        <p>
            <label for="password">Password:</label><br>
            <input type="password" id="password" name="password" required>
        </p>
        
        <p>
            <button type="submit">Log In</button>
        </p>
    </fieldset>
</form>

Question 12: Heading Hierarchy

Q: Explain the importance of proper heading hierarchy. Fix the following incorrect heading structure:

<h1>Website Title</h1>
<h3>Section 1</h3>
<h2>Subsection 1.1</h2>
<h1>Section 2</h1>

A:

Importance:

  • Screen readers use headings to navigate the page
  • Search engines use heading structure to understand content
  • Headings create a logical document outline
  • Should be sequential (don't skip levels)
  • Only one h1 per page (the main title)

Corrected structure:

<h1>Website Title</h1>
<h2>Section 1</h2>
<h3>Subsection 1.1</h3>
<h2>Section 2</h2>

Frequently Asked Questions

Q: Can I use HTML attributes like "border" on tables?
A: Yes, basic HTML attributes are allowed. The restriction is on CSS styling.
Q: Is it okay to use the "width" attribute on images?
A: Yes, width and height attributes are acceptable for images.
Q: Can I use <strong> and <em> for emphasis?
A: Absolutely! These are semantic HTML tags and should be used appropriately.
Q: What if I need to align text or elements?
A: Use semantic HTML structure instead. For example, use <blockquote> for indented quotes rather than CSS padding.
Q: Can I include an image of myself instead of Don Norman?
A: Yes, but the assignment content should still demonstrate HCI concepts. You can create a portfolio site about yourself.
Q: How many commits should I make to GitHub?
A: There's no specific number, but commit frequently (at least 10-15 commits). Each time you complete a page or significant section, commit with a descriptive message.
